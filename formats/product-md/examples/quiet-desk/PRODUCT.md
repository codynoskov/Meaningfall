# Quiet Desk

## Overview

Quiet Desk is a simple public product guidance file for a personal productivity landing page aimed at independent consultants.

The first version is body-only on purpose. The product owner has not yet selected durable frontmatter facts beyond the working product idea, and there is no confirmed map, state model, area model, or action list that would improve generation quality.

The page should stay small and direct. It should explain the value of a calmer consulting workspace, invite interested people to make contact, and avoid implying that a full SaaS app, account system, billing flow, documentation site, or automation product already exists.

### Interpretation

Agents should treat this file as product orientation, not as a schema to complete. If a later pass adds frontmatter, it should add only facts the product owner confirms or accepts, such as the public name, contact address, simple web context, home item, or a request-contact action.

### Boundaries

Do not invent authentication, pricing, dashboards, templates, onboarding, integrations, or system items for this first version. Those may become real product structure later, but they are not implied by the landing page.

### Validation And Feedback

If the landing page eventually includes a contact form, generated files should handle required values, invalid email addresses, submission failure, and successful receipt. Those are downstream form and implementation responsibilities, not ProductMD frontmatter facts yet.
