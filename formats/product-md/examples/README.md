# ProductMD Examples

Status: current example suite.

This folder contains ProductMD examples outside the normative specification.
## Complete Examples

Complete examples are full `PRODUCT.md` files that can be copied into a project root. They show how ProductMD frontmatter and Markdown body guidance work together at different levels of product pressure.

In structured examples, `##` body headings normally align with major ProductMD owners such as `Profile`, `Entities`, `Conditions`, `Actions`, `Map`, and `Mechanics`. `## Overview` is the main body-only exception because it gives connected product framing rather than compact facts. Optional explanations use `###` headings inside the nearest owner.

Current complete examples:

- [`quiet-desk/PRODUCT.md`](quiet-desk/PRODUCT.md) - body-only ProductMD guidance for a simple landing page before structured facts are selected.
- [`signal-lantern/PRODUCT.md`](signal-lantern/PRODUCT.md) - compact ProductMD structure for a small incident communication product.
- [`fieldnote/PRODUCT.md`](fieldnote/PRODUCT.md) - source-backed profile with intentional `TBD` values and no forced map, conditions, or actions.
- [`field-notes-library/PRODUCT.md`](field-notes-library/PRODUCT.md) - content library with collection entities and durable article-reading structure.
- [`lumen-docs/PRODUCT.md`](lumen-docs/PRODUCT.md) - documentation product with product-map structure, catalog-form compositions, and named navigations, including an override that hides archived material from ordinary guide navigation.
- [`harbor-hotel/PRODUCT.md`](harbor-hotel/PRODUCT.md) - booking surface with nested booking states and an embedded provider boundary.
- [`field-market/PRODUCT.md`](field-market/PRODUCT.md) - marketplace with public, seller, admin, commerce, policy, and payment-recovery pressure.
- [`ops-console/PRODUCT.md`](ops-console/PRODUCT.md) - internal operations console with semantic overlay composition, a workload relationship surface, non-visual system actions, and operational mechanics.
- [`case-brief/PRODUCT.md`](case-brief/PRODUCT.md) - AI-assisted support workflow with human review pressure and AI assistance mechanics.
- [`carebridge/PRODUCT.md`](carebridge/PRODUCT.md) - regulated intake and care coordination example with sensitive legal/review guidance.

The public example surface points readers to complete `PRODUCT.md` files.
